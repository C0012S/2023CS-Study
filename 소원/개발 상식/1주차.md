# ✅좋은 코드란 무엇인가?

## 읽기 쉬운 코드

### 일관성이 있는 네이밍

- 변수명 뿐만 아니라 함수명에 의해서도 규칙이 있으면 일관성을 지킬 수 있다

### 일관성이 있는 들여쓰기

- 협업하는 상대와의 룰을 지켜 들여쓰기를 하면 좋다

## 중복이 제거된 코드

- 동일한 로직을 수행하는 코드가 있다면 추출을 하게 될 것
- 하지만 단순히 중복된 코드라서 추출을 해낸다면 다른 곳에서 재사용 하기 어렵다
- 추출이 아닌 추상화를 한다
    - 추출: 특별한 기준 없이 무작정 밖으로 빼내는 것
    - 추상화: 일정한 기준을 적용시켜 정리해낸 것
